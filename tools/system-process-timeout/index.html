Parameters

Name Type Required Default Description Aliases
foreground number No - when not running timeout directly from a shell prompt, allow COMMAND to read from the TTY and get TTY signals; in this mode, children of COMMAND will not be timed out -f, --foreground
kill-after string No - also send a KILL signal if COMMAND is still running this long after the initial signal was sent -k, --kill-after
preserve-status string No - exit with the same status as COMMAND, even when the command times out -p, --preserve-status
signal array No - specify the signal to be sent on timeout; SIGNAL may be a name like 'HUP' or a number; see 'kill -l' for a list of signals -s, --signal
verbose number No - diagnose to standard error any signal sent upon timeout --help display this help and exit --version output version information and exit -v, --verbose
